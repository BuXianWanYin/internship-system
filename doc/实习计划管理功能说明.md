# 实习计划管理功能说明文档

## 一、功能概述

**实习计划管理**是高校实习管理系统的核心功能模块之一，主要用于学校管理员和系统管理员制定、审核、发布和管理学生的实习计划。实习计划是整个实习活动的指导性文件，明确了实习的目标、要求、时间安排和考核标准。

## 二、功能定位

实习计划管理在整个实习流程中的位置：

```
实习计划管理 → 岗位发布 → 双选管理 → 实习过程跟踪 → 实习评价
     ↓              ↓          ↓            ↓            ↓
  制定计划      企业发布    学生申请    日志/周报    最终评价
  审核发布      岗位审核    企业筛选    成果提交    成绩评定
```

**实习计划是实习活动的起点**，只有发布了实习计划，学生才能开始申请实习，企业才能发布对应的实习岗位。

## 三、主要功能

### 3.1 创建实习计划

**功能描述：** 学校管理员或系统管理员可以创建新的实习计划。

**包含内容：**
- **基本信息**
  - 计划名称（如："2024年春季生产实习计划"）
  - 计划编号（唯一标识，如："PLAN2024001"）
  - 关联学期（选择对应的学期）
  - 适用范围（学校/学院/专业）
  
- **实习类型**
  - 生产实习
  - 毕业实习
  - 认知实习
  - 其他类型

- **时间安排**
  - 实习开始日期
  - 实习结束日期

- **实习内容**（支持富文本编辑）
  - **实习大纲**：说明本次实习的目的、意义、总体安排
  - **任务要求**：明确学生在实习期间需要完成的具体任务
  - **考核标准**：规定实习成绩的评定标准和权重

**使用场景：**
- 每学期开始前，学校管理员需要为不同专业制定实习计划
- 系统管理员可以为多个学校创建统一的实习计划模板

### 3.2 编辑实习计划

**功能描述：** 修改已创建的实习计划信息。

**限制条件：**
- 只有**草稿状态**的实习计划可以编辑
- 已提交审核、已发布、已拒绝的计划不能直接编辑
- 如需修改已发布的计划，需要先撤销发布或创建新版本

### 3.3 查询实习计划列表

**功能描述：** 分页查询实习计划，支持多条件筛选。

**筛选条件：**
- 计划名称（模糊搜索）
- 学期
- 学校
- 学院
- 专业
- 状态（草稿/待审核/已通过/已拒绝/已发布）

**显示信息：**
- 计划名称、计划编号
- 学期、学校、学院、专业
- 实习类型、开始日期、结束日期
- 状态、创建时间

### 3.4 实习计划审批流程

**流程说明：**

```
草稿(0) → 提交审核 → 待审核(1) → 审核 → 已通过(2) → 发布 → 已发布(4)
                              ↓
                           已拒绝(3)
```

**状态说明：**

| 状态 | 状态码 | 说明 | 可执行操作 |
|------|--------|------|-----------|
| 草稿 | 0 | 刚创建或保存的未提交计划 | 编辑、提交审核、删除 |
| 待审核 | 1 | 已提交等待审核的计划 | 审核（通过/拒绝） |
| 已通过 | 2 | 审核通过但未发布的计划 | 发布、查看详情 |
| 已拒绝 | 3 | 审核被拒绝的计划 | 查看详情、重新编辑（需重新创建） |
| 已发布 | 4 | 已发布可供学生使用的计划 | 查看详情、撤销发布（可选） |

**审批操作：**
- **提交审核**：将草稿状态的计划提交给上级审核
- **审核通过**：审核人填写审核意见，将计划状态改为"已通过"
- **审核拒绝**：审核人填写拒绝原因，将计划状态改为"已拒绝"

### 3.5 发布实习计划

**功能描述：** 将已通过审核的实习计划发布，使其对学生和企业可见。

**发布后的效果：**
- 学生可以看到该实习计划，并开始申请实习
- 企业可以发布与该计划相关的实习岗位
- 系统开始记录该计划的实习活动

**限制条件：**
- 只有**已通过审核**状态的计划才能发布
- 已发布的计划不能直接修改（需要撤销发布或创建新版本）

### 3.6 删除实习计划

**功能描述：** 删除不需要的实习计划（软删除）。

**限制条件：**
- 只有**草稿状态**的计划可以删除
- 已发布的计划不能删除（保护历史数据）

### 3.7 历史计划复用（规划功能）

**功能描述：** 基于历史实习计划创建新计划，提高工作效率。

**使用场景：**
- 每年/每学期的实习计划内容相似
- 可以复制历史计划，修改时间、学期等信息后快速创建新计划

## 四、权限控制

### 4.1 可访问角色

- **ROLE_SYSTEM_ADMIN**（系统管理员）
  - 可以管理所有学校的实习计划
  - 可以查看、创建、编辑、审核、发布所有计划

- **ROLE_SCHOOL_ADMIN**（学校管理员）
  - 只能管理本学校的实习计划
  - 可以查看、创建、编辑、提交审核本学校的计划
  - 可以审核本学校的计划（如果配置了审核权限）

### 4.2 权限说明

- **查看权限**：只能查看自己权限范围内的计划
  - 系统管理员：所有计划
  - 学校管理员：本学校的计划

- **创建权限**：系统管理员和学校管理员都可以创建

- **编辑权限**：只能编辑自己创建的、且状态为草稿的计划

- **审核权限**：需要配置审核权限，通常由上级管理员或指定审核人执行

- **发布权限**：通常由学校管理员或系统管理员执行

## 五、数据字段说明

### 5.1 基本信息字段

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| planId | Long | 计划ID（主键，自增） | 否 |
| planName | String | 实习计划名称 | 是 |
| planCode | String | 实习计划编号（唯一） | 是 |
| semesterId | Long | 关联学期ID | 否 |
| schoolId | Long | 所属学校ID | 是 |
| collegeId | Long | 所属学院ID（NULL表示全校） | 否 |
| majorId | Long | 所属专业ID（NULL表示全院） | 否 |
| planType | String | 实习类型 | 是 |

### 5.2 时间字段

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| startDate | LocalDate | 实习开始日期 | 是 |
| endDate | LocalDate | 实习结束日期 | 是 |

### 5.3 内容字段（富文本）

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| planOutline | String | 实习大纲 | 否 |
| taskRequirements | String | 任务要求 | 否 |
| assessmentStandards | String | 考核标准 | 否 |

### 5.4 状态和审核字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| status | Integer | 状态：0-草稿，1-待审核，2-已通过，3-已拒绝，4-已发布 |
| auditUserId | Long | 审核人ID |
| auditTime | LocalDateTime | 审核时间 |
| auditOpinion | String | 审核意见 |
| publishTime | LocalDateTime | 发布时间 |
| createUserId | Long | 创建人ID |

## 六、使用场景示例

### 场景一：新学期开始前制定实习计划

1. **学校管理员登录系统**
2. **创建实习计划**
   - 填写计划名称："2024年春季计算机专业生产实习计划"
   - 选择学期："2023-2024学年第二学期"
   - 选择适用范围：计算机学院 > 计算机科学与技术专业
   - 设置实习时间：2024年3月1日 - 2024年6月30日
   - 编写实习大纲、任务要求、考核标准
3. **保存为草稿**，可以随时修改
4. **提交审核**，等待上级审核
5. **审核通过后发布**，学生可以看到该计划并开始申请

### 场景二：审核实习计划

1. **审核人登录系统**
2. **查看待审核的实习计划列表**
3. **点击查看详情**，仔细审查计划内容
4. **审核操作**：
   - 如果内容符合要求，点击"审核通过"，填写审核意见
   - 如果内容有问题，点击"审核拒绝"，填写拒绝原因
5. **审核通过后**，创建人可以发布计划

### 场景三：学生查看实习计划

1. **学生登录系统**
2. **在首页或实习管理模块**查看已发布的实习计划
3. **查看计划详情**，了解：
   - 实习时间安排
   - 实习任务要求
   - 考核标准
4. **根据计划要求**准备实习申请材料

## 七、与其他模块的关系

### 7.1 与岗位管理的关系

- 实习计划发布后，企业可以发布与该计划相关的实习岗位
- 岗位需要关联到具体的实习计划
- 学生只能申请与已发布实习计划相关的岗位

### 7.2 与双选管理的关系

- 实习计划是双选活动的前提
- 学生申请实习时，需要选择对应的实习计划
- 双选结果需要符合实习计划的时间安排

### 7.3 与过程跟踪的关系

- 实习计划规定了实习的时间范围
- 学生的实习日志、周报需要在计划的时间范围内提交
- 考核标准来自实习计划

### 7.4 与评价管理的关系

- 实习计划的考核标准是评价学生实习成绩的依据
- 最终评价需要参考计划中规定的各项指标

## 九、注意事项

1. **计划编号唯一性**：每个实习计划必须有唯一的计划编号，系统会自动校验
2. **时间合理性**：实习开始日期必须早于结束日期
3. **适用范围**：计划可以设置为全校、全院或特定专业
4. **状态流转**：计划状态必须按照规定的流程流转，不能跳过中间状态
5. **权限控制**：不同角色只能操作权限范围内的计划
6. **数据保护**：已发布的计划不能随意删除，保护历史数据完整性

## 十、未来扩展功能

1. **计划模板**：提供常用实习计划模板，快速创建
2. **批量操作**：支持批量创建、批量审核、批量发布
3. **计划对比**：对比不同学期或不同专业的实习计划
4. **统计分析**：统计实习计划的执行情况、完成率等
5. **消息通知**：计划发布后自动通知相关学生和教师
6. **版本管理**：支持计划版本管理，可以查看历史版本

## 十一、总结

实习计划管理是实习管理系统的**基础模块**，它为整个实习活动提供了**指导框架**。通过实习计划，可以：

- ✅ **规范实习流程**：明确实习的时间、任务、要求
- ✅ **统一考核标准**：确保评价的公平性和一致性
- ✅ **提高管理效率**：集中管理，便于审核和发布
- ✅ **支持决策分析**：为实习管理提供数据支持

只有制定了完善的实习计划，才能确保实习活动的有序进行和高质量完成。
