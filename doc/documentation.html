📋 项目概述

项目名称：高校实习过程跟踪与评价管理系统

开发模式：前后端分离

维护者：邓梓伶

完成时间：2025年12月

开发进度：已完成

项目简介

高校实习过程跟踪与评价管理系统是一个面向高校、企业、学生的综合性实习管理平台。系统实现了从实习计划制定、岗位发布、申请审核、面试安排、考勤管理、日志周报、成果反馈到综合评价的完整实习管理流程。

系统采用现代化的前后端分离架构，基于Spring Boot和Vue 3技术栈开发，提供了完整的实习管理解决方案。系统支持多种实习类型（合作企业实习、自主实习），实现了多角色协同工作，为高校实习管理提供了数字化、系统化的管理工具。

项目背景

随着高等教育的不断发展，实习已成为高校人才培养的重要环节。然而，传统的实习管理方式存在诸多问题：

  • 管理效率低下：传统的纸质文档管理方式，信息传递慢，容易丢失
  • 信息不透明：学生、教师、企业之间信息沟通不畅，缺乏统一的信息平台
  • 流程不规范：实习申请、审核、评价等流程缺乏标准化管理
  • 数据分散：实习相关数据分散在各个部门，难以统一管理和分析
  • 评价体系不完善：缺乏科学的评价标准和量化指标

因此，开发一套完整的实习管理系统，实现实习管理的数字化、系统化、规范化，具有重要的现实意义。

项目目标

  • 提高管理效率：通过数字化管理，提高实习管理效率，减少人工操作
  • 规范管理流程：建立标准化的实习管理流程，确保各个环节规范有序
  • 促进校企合作：为校企合作提供便捷的管理平台，促进校企深度合作
  • 完善评价体系：建立科学的评价体系，实现多维度、量化的实习评价
  • 数据统计分析：提供丰富的数据统计和分析功能，为决策提供数据支撑

项目特点

  • 多角色权限管理：支持学生、教师、企业、企业导师、学校管理员等多种角色，每个角色拥有不同的权限和功能
  • 完整的实习流程：涵盖实习申请、面试、考勤、日志、周报、成果、评价全流程，实现实习全生命周期管理
  • 校企协同：支持校企合作申请、审核、管理等功能，促进校企深度合作
  • 数据可视化：提供丰富的统计图表和数据分析功能，直观展示实习数据
  • 移动端适配：响应式设计，支持移动端访问，方便学生随时查看和操作
  • 灵活配置：支持系统参数配置，可根据不同学校的需求进行定制
  • 安全可靠：采用JWT认证、数据权限控制等安全机制，保证数据安全

适用场景

  • 高校实习管理：适用于各类高校的实习管理工作
  • 校企合作管理：适用于校企合作项目的管理和跟踪
  • 学生实习跟踪：适用于学生实习过程的跟踪和评价
  • 数据统计分析：适用于实习数据的统计分析和报表生成

🚀 快速启动

环境要求

  • JDK：1.8 或以上
  • Maven：3.x
  • Node.js：16.0.0 或以上
  • npm：7.0.0 或以上
  • MySQL：8.0 或以上
  • Redis：3.x 或以上

后端启动

1. 数据库准备

  1. 确保MySQL服务已启动（版本8.0或以上）
  2. 创建数据库：
    CREATE DATABASE internship_management_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  3. 导入SQL文件：执行 internship-server/sql/internship_management_db.sql 文件

    注意：SQL文件包含完整的数据库结构和初始数据，导入前请确保数据库为空。

2. Redis准备

  1. 确保Redis服务已启动（版本3.x或以上）
  2. 默认配置：localhost:6379，无密码
  3. 如需修改，请在配置文件中更新Redis连接信息

3. 配置文件修改

修改 internship-server/src/main/resources/application.yml 文件：

  • 数据库配置：修改数据库连接地址、用户名、密码
  • Redis配置：修改Redis连接地址、端口、密码（如有）
  • JWT配置：可修改Token有效期（默认7天）
  • 文件上传配置：可修改文件上传路径和最大文件大小
# 数据库配置示例 spring: datasource: url: jdbc:mysql://localhost:3306/internship_management_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai username: root password: 123456 # Redis配置示例 spring: redis: host: localhost port: 6379 password:

4. 编译和启动

  1. 进入后端项目目录：cd internship-server
  2. 编译项目：mvn clean install
  3. 启动项目：
    • 方式一：运行 InternshipServerApplication 主类
    • 方式二：使用Maven命令：mvn spring-boot:run
    • 方式三：打包后运行：java -jar target/internship-server-0.0.1-SNAPSHOT.jar
  4. 验证启动：访问 http://localhost:8081/swagger-ui/index.html，如能正常打开API文档，说明启动成功

5. 常见问题

端口被占用：如果8081端口被占用，可在application.yml中修改server.port配置

数据库连接失败：检查数据库服务是否启动，连接信息是否正确

Redis连接失败：检查Redis服务是否启动，连接信息是否正确（Redis非必需，但建议配置）

前端启动

1. 环境准备

  • 确保Node.js版本 >= 16.0.0
  • 确保npm版本 >= 7.0.0（或使用yarn）
  • 建议使用npm国内镜像源（如淘宝镜像）以提高下载速度

2. 安装依赖

  1. 进入前端项目目录：cd internship-web
  2. 安装依赖：npm install 或 yarn install

    提示：如果安装速度慢，可以使用国内镜像：npm install --registry=https://registry.npmmirror.com

3. 环境变量配置

在 internship-web 目录下创建 .env 文件（如果不存在）：

# 前端服务端口 VITE_PORT=5173 # 后端API地址 VITE_API_BASE_URL=http://localhost:8081

重要：必须配置VITE_API_BASE_URL，否则前端无法连接后端API。如果后端运行在其他端口，请相应修改。

4. 启动开发服务器

  1. 启动开发服务器：npm run dev 或 yarn dev
  2. 等待编译完成，看到类似以下输出表示启动成功：
    VITE v6.0.0 ready in 500 ms ➜ Local: http://localhost:5173/ ➜ Network: use --host to expose
  3. 访问前端应用：http://localhost:5173

5. 生产环境构建

如需构建生产版本：

  1. 执行构建命令：npm run build 或 yarn build
  2. 构建产物在 dist 目录下
  3. 可以使用 npm run preview 预览生产构建

6. 常见问题

依赖安装失败：清除缓存后重新安装：npm cache clean --force && rm -rf node_modules package-lock.json && npm install

端口被占用：修改.env文件中的VITE_PORT配置

API请求失败：检查.env文件中的VITE_API_BASE_URL配置是否正确，确保后端服务已启动

编译错误：检查Node.js版本是否符合要求，清除node_modules后重新安装

Docker启动（可选）

注意：当前项目暂未提供Docker配置，如需使用Docker部署，需要自行编写Dockerfile和docker-compose.yml文件。

🏗️ 系统架构

整体架构

系统采用前后端分离架构，前端使用Vue 3 + Vite构建，后端使用Spring Boot提供RESTful API服务。

架构模式：前后端分离 + RESTful API

通信协议：HTTP/HTTPS

数据格式：JSON

认证方式：JWT Token

请求流程

系统采用标准的MVC架构模式，请求处理流程如下：

  1. 用户请求：用户通过浏览器访问前端应用，触发操作
  2. 前端处理：前端Vue组件通过Axios发送HTTP请求到后端API
  3. 请求拦截：后端Spring Security拦截请求，验证JWT Token的有效性
  4. 权限验证：通过Token验证后，检查用户是否有权限访问该接口（@PreAuthorize）
  5. Controller层：请求进入对应的Controller，进行参数验证和格式转换
  6. Service层：Controller调用Service层处理业务逻辑，Service层负责事务管理
  7. Mapper层：Service层通过MyBatis Plus的Mapper接口访问数据库
  8. 数据返回：数据库查询结果经过Service层处理后，返回给Controller
  9. 响应封装：Controller将结果封装成统一的Result格式返回前端
  10. 前端更新：前端接收数据，更新Pinia状态，触发界面重新渲染

关键点：

  • 所有API请求都需要携带JWT Token（登录接口除外）
  • Token过期后需要重新登录或刷新Token
  • 权限验证在Controller方法级别进行，确保数据安全
  • 所有响应都使用统一的Result格式，便于前端统一处理

后端结构

后端采用经典的分层架构，各层职责明确：

  • Controller层：接收HTTP请求，进行参数验证和格式转换，调用Service层处理业务逻辑，返回统一格式的响应
  • Service层：核心业务逻辑处理，事务管理，数据权限控制，调用Mapper层进行数据操作
  • Mapper层：数据访问层，使用MyBatis Plus进行数据库操作，支持复杂查询和动态SQL
  • Domain层：实体类（Entity）和数据传输对象（DTO），定义数据结构和业务对象
  • Common层：公共组件，包括工具类、配置类、常量类、枚举类、异常处理、统一响应等
  • Security层：安全认证和权限控制，包括JWT过滤器、用户认证服务、权限验证等

设计原则：

  • 单一职责：每层只负责自己的职责，不越界
  • 依赖倒置：上层依赖下层接口，不依赖具体实现
  • 开闭原则：对扩展开放，对修改关闭
  • 事务管理：在Service层进行事务管理，保证数据一致性

前端结构

前端采用Vue 3 + Composition API的组件化开发模式：

  • Views：页面组件，按角色（admin、student、teacher、enterprise）和模块划分，每个页面对应一个路由
  • Components：可复用组件，包括业务组件（如评价表单）和通用组件（如布局、图表、统计卡片等）
  • Router：路由配置，按模块划分，支持路由守卫进行权限控制
  • Store：状态管理，使用Pinia进行全局状态管理，主要包括用户信息、权限信息等
  • API：接口定义，按模块划分，统一封装HTTP请求，处理错误和响应
  • Utils：工具函数，包括请求封装、权限验证、日期处理、数据导出、状态转换等
  • Config：配置文件，包括菜单配置、系统配置等

设计特点：

  • 组件化开发：提高代码复用性和可维护性
  • 响应式设计：支持PC端和移动端自适应
  • 路由守卫：实现基于角色的权限控制
  • 统一请求封装：统一处理错误、loading、Token刷新等
  • 状态管理：使用Pinia进行状态管理，替代Vuex

🔄 核心业务流程

实习申请流程

系统支持两种类型的实习申请：合作企业实习和自主实习。两种类型的流程有所不同。

合作企业实习流程

  1. 准备阶段：
    • 系统管理员/学校管理员创建学校、学院、专业、班级等基础数据
    • 企业注册账号，提交企业信息，等待审核
    • 管理员审核企业注册，建立校企合作关系
  2. 计划阶段：
    • 管理员/教师创建实习计划（草稿状态）
    • 提交实习计划审核，审核通过后发布计划
    • 企业创建实习岗位（草稿状态）
    • 提交岗位审核，审核通过后发布岗位
  3. 申请阶段：
    • 学生浏览已发布的实习岗位
    • 学生选择目标岗位，填写申请信息（包括简历、申请理由等），提交申请
    • 申请状态：待审核（0）
  4. 审核阶段：
    • 学校管理员/学院领导/班主任审核学生申请
    • 审核通过：状态变为已通过（1），进入企业筛选环节
    • 审核拒绝：状态变为已拒绝（2），流程结束
  5. 筛选阶段：
    • 企业管理员查看申请列表，进行筛选操作
    • 可以标记感兴趣、安排面试、录用或拒绝
    • 如果安排面试，创建面试记录
  6. 面试阶段（可选）：
    • 企业安排面试，设置面试时间、地点、类型（线上/线下）
    • 学生确认面试时间
    • 企业完成面试后，录入面试结果（通过/未通过/待定）
    • 根据面试结果决定是否录用
  7. 录用阶段：
    • 企业管理员录用学生，状态变为已录用（3）
    • 学生确认上岗，开始实习
    • 企业管理员为学生分配企业导师
  8. 实习阶段：
    • 学生进行实习，提交考勤、日志、周报、成果等
    • 企业导师和教师进行审核和评价
  9. 结束阶段：
    • 实习结束，提交各类评价
    • 系统自动计算综合评分

自主实习流程

  1. 申请阶段：
    • 学生提交自主实习申请，填写实习单位信息、实习内容等
    • 申请状态：待审核（0）
  2. 审核阶段：
    • 班主任审核学生申请
    • 审核通过：状态变为已通过（1），系统自动确认上岗
    • 审核拒绝：状态变为已拒绝（2），流程结束
  3. 实习阶段：
    • 学生进行实习，提交日志、周报、成果等
    • 班主任进行审核和评价
  4. 结束阶段：
    • 实习结束，提交自我评价和学校评价
    • 系统自动计算综合评分

考勤管理流程

考勤管理是实习管理的重要环节，系统提供了完整的考勤管理功能，包括考勤组配置、考勤记录、考勤统计等。

考勤组配置流程

  1. 创建考勤组：
    • 企业导师/企业管理员创建考勤组
    • 设置考勤组名称、描述、工作日类型（周一到周五/周一到周六/周一到周日/自定义）
    • 设置默认工作时长（小时）
  2. 配置考勤规则：
    • 为考勤组添加考勤规则
    • 设置规则名称、规则类型（签到时间、签退时间、工作时长等）
    • 设置规则条件（如：迟到时间阈值、早退时间阈值等）
  3. 配置时间段：
    • 为考勤组添加时间段
    • 设置时间段名称、开始时间、结束时间
    • 一个考勤组可以有多个时间段（如：上午、下午）
  4. 分配学生：
    • 将学生添加到考勤组
    • 可以选择多个学生批量添加
    • 学生可以属于多个考勤组

考勤记录流程

  1. 学生签到/签退：
    • 学生每日进行签到和签退操作
    • 系统记录签到时间、签退时间
    • 系统自动计算工作时长
    • 系统根据考勤规则判断考勤类型（出勤/迟到/早退）
  2. 请假申请：
    • 学生可以提交请假申请
    • 填写请假日期、请假类型（事假/病假/调休等）、请假原因
    • 申请状态：待确认（0）
  3. 考勤确认：
    • 企业导师/企业管理员查看考勤记录
    • 对考勤记录进行确认（已确认/已拒绝）
    • 可以填写确认意见
    • 确认后，考勤记录状态变为已确认（1）
  4. 考勤统计：
    • 系统自动统计考勤数据
    • 提供考勤统计报表，包括出勤率、迟到次数、早退次数、请假次数等
    • 支持按时间段、按学生、按考勤组等维度进行统计

考勤类型说明

考勤类型 代码 说明
出勤 1 正常出勤，按时签到签退
迟到 2 签到时间晚于规定时间
早退 3 签退时间早于规定时间
请假 4 已提交请假申请并获批
缺勤 5 未签到也未请假

评价管理流程

评价管理是实习管理的重要环节，系统建立了完善的评价体系，包括自我评价、企业评价、学校评价和综合评分。

评价体系说明

系统采用多维度评价体系，从不同角度对学生的实习表现进行评价：

  • 自我评价：学生对自己实习表现的自我评价，包括工作表现、学习收获、能力提升等
  • 企业评价：企业导师或企业管理员对学生的评价，从企业角度评价学生的实习表现
  • 学校评价：班主任或学校管理员对学生的评价，从学校角度评价学生的实习表现
  • 综合评分：系统根据三种评价的权重自动计算综合评分

评价流程

  1. 自我评价阶段：
    • 实习结束或即将结束时，学生提交自我评价
    • 填写评价内容：实习总结、工作表现自评、学习收获、能力提升、不足之处、改进方向等
    • 对各项指标进行评分（如：工作态度、专业技能、团队协作、创新能力等）
    • 提交后，评价状态为已评价
  2. 企业评价阶段：
    • 企业导师或企业管理员对学生进行评价
    • 填写评价内容：工作表现、专业技能、团队协作、学习能力、创新能力等
    • 对各项指标进行评分
    • 可以填写评价意见和建议
    • 提交后，评价状态为已评价
  3. 学校评价阶段：
    • 班主任或学校管理员对学生进行评价
    • 填写评价内容：实习态度、实习效果、能力提升等
    • 对各项指标进行评分
    • 可以填写评价意见
    • 提交后，评价状态为已评价
  4. 综合评分计算：
    • 系统根据系统配置的权重自动计算综合评分
    • 权重配置：自我评价权重、企业评价权重、学校评价权重（权重之和为100%）
    • 计算公式：综合评分 = 自我评价分数 × 自我评价权重 + 企业评价分数 × 企业评价权重 + 学校评价分数 × 学校评价权重
    • 综合评分自动保存到综合评分表
  5. 评价查看和统计：
    • 学生可以查看自己的所有评价和综合评分
    • 教师和管理员可以查看学生的评价情况
    • 系统提供评价统计图表，包括评价分数分布、评价趋势等
    • 支持导出评价报告

评价指标说明

系统支持多维度评价指标，主要包括：

  • 工作态度：工作积极性、责任心、主动性等
  • 专业技能：专业知识的应用、技能掌握程度等
  • 团队协作：与同事的协作能力、沟通能力等
  • 学习能力：学习新知识的能力、适应能力等
  • 创新能力：创新思维、解决问题的能力等
  • 实习效果：实习目标的达成情况、实习成果等

评价权重配置：系统管理员可以在系统配置中设置评价权重，默认权重为：自我评价30%、企业评价50%、学校评价20%。权重可以根据实际需求进行调整。

校企合作流程

  1. 企业提交合作申请
  2. 管理员审核合作申请
  3. 审核通过后建立合作关系
  4. 企业可以发布岗位、管理学生
  5. 管理员可以查看合作统计

🎓 答辩准备

常见问题

1. 项目背景和意义

问题：为什么选择开发这个系统？

回答要点：

  • 传统实习管理方式效率低下，信息不透明
  • 缺乏统一的管理平台，数据分散
  • 评价体系不完善，缺乏量化指标
  • 校企沟通不畅，协同效率低
  • 系统化、数字化管理是发展趋势

2. 技术选型原因

问题：为什么选择这些技术栈？

回答要点：

  • Spring Boot：成熟稳定，生态丰富，开发效率高
  • Vue 3：渐进式框架，性能优秀，学习曲线平缓
  • MyBatis Plus：简化CRUD操作，提高开发效率
  • JWT：无状态认证，适合前后端分离架构
  • Element Plus：组件丰富，文档完善，适合快速开发

3. 系统难点和解决方案

问题：开发过程中遇到的主要难点是什么？

回答要点：

  • 权限控制：使用Spring Security + JWT实现细粒度权限控制
  • 数据权限：根据角色动态过滤数据，实现数据隔离
  • 复杂查询：使用MyBatis Plus的QueryWrapper构建动态查询条件
  • 文件上传：实现文件上传、存储、下载功能，支持多种文件类型
  • 数据统计：使用ECharts实现数据可视化，支持多维度统计

技术亮点

  • 前后端分离架构：提高开发效率，便于维护和扩展
  • JWT无状态认证：支持分布式部署，提高系统可扩展性
  • RBAC权限模型：灵活的权限控制，支持多角色多权限
  • 数据权限控制：根据角色动态过滤数据，保证数据安全
  • 响应式设计：支持PC端和移动端，提升用户体验
  • 数据可视化：丰富的图表展示，直观的数据分析
  • Excel导入导出：支持批量数据导入导出，提高操作效率

难点解决

1. 复杂业务逻辑处理

采用分层架构，将复杂业务逻辑拆分到Service层，使用事务管理保证数据一致性。

2. 权限控制

使用Spring Security的@PreAuthorize注解实现方法级权限控制，结合JWT Token实现无状态认证。

3. 数据统计和报表

使用MyBatis的复杂查询和聚合函数，结合ECharts实现多维度数据统计和可视化展示。

接口规范

  • 请求方式：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）

  • 状态码：200（成功）、400（参数错误）、401（未认证）、403（无权限）、500（服务器错误）
  • 分页参数：page（页码）、size（每页数量）
  • 排序参数：sort（排序字段）、order（排序方式：asc/desc）
