# 面试模块操作说明

## 模块概述

面试模块用于管理学生的面试信息，包括面试的创建、确认、结果录入等流程。面试是合作企业实习申请流程中的一个重要环节。

## 一、面试状态说明

| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| 0 | 待确认 | 企业已安排面试，等待学生确认 |
| 1 | 已确认 | 学生已确认面试时间 |
| 2 | 已完成 | 面试已完成，等待录入结果 |
| 3 | 已取消 | 面试已取消 |

## 二、操作流程

### 2.1 创建面试

#### 步骤1：企业安排面试
- **操作角色**：企业管理员
- **操作接口**：`POST /internship/interview`
- **前置条件**：
  - 学生申请状态为"已通过"
  - 企业已通过筛选操作选择"安排面试"
- **操作说明**：
  1. 企业管理员查看申请详情
  2. 点击"安排面试"
  3. 填写面试信息：
     - 面试类型（现场面试/视频面试/电话面试）
     - 面试时间
     - 面试地点（现场面试需要）
     - 面试链接（视频面试需要）
     - 联系电话
     - 备注信息
  4. 提交面试安排
- **状态**：`待确认 (0)`

### 2.2 学生确认面试

#### 步骤1：查看面试通知
- **操作角色**：学生
- **操作说明**：
  1. 学生登录系统
  2. 查看面试通知
  3. 查看面试详情（时间、地点、类型等）

#### 步骤2：确认或拒绝面试
- **操作角色**：学生
- **操作接口**：`POST /internship/interview/{interviewId}/confirm`
- **前置条件**：
  - 面试状态为"待确认"
- **操作说明**：
  1. 学生查看面试详情
  2. 选择操作：
     - **确认面试**：状态变为 `已确认 (1)`
     - **拒绝面试**：可以填写拒绝原因，面试可能被取消
  3. 提交确认
- **状态流转**：`待确认 (0)` → `已确认 (1)`

### 2.3 录入面试结果

#### 步骤1：完成面试
- **操作角色**：企业管理员、企业导师
- **操作说明**：
  1. 面试完成后，企业相关人员登录系统
  2. 进入面试管理页面
  3. 找到对应的面试记录

#### 步骤2：录入面试结果
- **操作角色**：企业管理员、企业导师
- **操作接口**：`POST /internship/interview/{interviewId}/result`
- **前置条件**：
  - 面试状态为"已确认"或"已完成"
- **操作说明**：
  1. 填写面试结果：
     - 面试结果（通过/不通过）
     - 面试评分（可选）
     - 面试评价
     - 备注信息
  2. 提交结果
  3. 状态变为"已完成"
  4. 如果面试通过，可以进一步操作（录用/拒绝）
- **状态流转**：`已确认 (1)` → `已完成 (2)`

### 2.4 取消面试

- **操作角色**：企业管理员、学生
- **操作接口**：`POST /internship/interview/{interviewId}/cancel`
- **前置条件**：
  - 面试状态为"待确认"或"已确认"
- **操作说明**：
  1. 填写取消原因
  2. 提交取消申请
  3. 状态变为"已取消"
- **状态流转**：→ `已取消 (3)`

## 三、面试类型说明

### 3.1 现场面试
- **说明**：学生到企业现场进行面试
- **需要信息**：面试地点、面试时间

### 3.2 视频面试
- **说明**：通过视频会议进行面试
- **需要信息**：面试链接、面试时间

### 3.3 电话面试
- **说明**：通过电话进行面试
- **需要信息**：联系电话、面试时间

## 四、权限说明

### 4.1 企业管理员
- 创建面试安排
- 录入面试结果
- 取消面试
- 查看本企业的所有面试记录

### 4.2 企业导师
- 录入面试结果（如果被授权）
- 查看分配给自己的学生的面试记录

### 4.3 学生
- 查看自己的面试通知
- 确认或拒绝面试
- 取消面试（在允许的情况下）

### 4.4 学校管理员
- 查看所有面试记录（用于监督和管理）

## 五、API接口列表

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/internship/interview` | POST | 创建面试 | 企业管理员 |
| `/internship/interview` | PUT | 更新面试 | 企业管理员 |
| `/internship/interview/{interviewId}` | GET | 查询面试详情 | 多角色 |
| `/internship/interview/page` | GET | 分页查询面试列表 | 多角色 |
| `/internship/interview/{interviewId}/confirm` | POST | 确认面试 | 学生 |
| `/internship/interview/{interviewId}/result` | POST | 录入面试结果 | 企业管理员/企业导师 |
| `/internship/interview/{interviewId}/cancel` | POST | 取消面试 | 企业管理员/学生 |
| `/internship/interview/{interviewId}` | DELETE | 删除面试 | 企业管理员 |

## 六、常见问题

### Q1: 面试可以修改时间吗？
A: 可以，企业管理员可以修改面试信息，但需要学生重新确认。

### Q2: 学生拒绝面试会怎样？
A: 学生拒绝面试后，企业可以取消面试或重新安排。

### Q3: 面试结果录入后可以修改吗？
A: 需要根据系统配置确定，一般录入后不能修改。

### Q4: 面试通过后会自动录用吗？
A: 不会，面试通过后，企业还需要进行录用操作。

### Q5: 可以安排多次面试吗？
A: 可以，企业可以为同一个申请安排多次面试。

## 七、注意事项

1. **时间安排**：面试时间要合理，不能早于当前时间。
2. **通知及时性**：企业安排面试后，系统会通知学生，学生应及时查看和确认。
3. **结果录入**：面试完成后应及时录入结果，避免影响后续流程。
4. **数据权限**：企业管理员只能操作本企业的面试记录。
5. **状态流转**：面试状态必须按照规定的流程流转。
